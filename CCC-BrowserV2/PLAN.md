# RPA自动化系统开发项目计划

## 项目概览

**项目名称**: 极简配置驱动RPA自动化系统
**项目目录**: `/Users/fcj/workspace/AI_SW/ccclubs_AI_RPA/CCC-BrowserV2`
**项目类型**: WEB全栈项目 (桌面客户端 + 后端服务 + 前端可视化)
**目标平台**: Windows / macOS / Ubuntu 三端桌面客户端

## 核心业务定位

**业务链路**: 首次沙箱登录 → 会话缓存复用 → 场景切换 → 后端API批量数据获取 → 循环自动填表单+提交 → 任务结束

**核心特性**:
- 流程完全配置化,零硬编码
- 一次登录、全程复用会话
- 多场景页面模板切换
- 批量数据来自后端业务API
- 全浏览器真人仿真、WZWS全维度绕过
- 全功能Web可视化
- 跨三端桌面客户端

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 桌面端壳 | Tauri | v2.0.0-beta.28 |
| 自动化内核 | Playwright-Core | 1.45.3 |
| 后端框架 | NestJS + Node.js | 10.3.0 + 20.12.2 |
| 前端框架 | Vue3 + Vite + Element Plus | 3.4.15 + 5.1.0 + 2.4.4 |
| 任务队列 | BullMQ + Redis | 5.12.0 + 7.0.15 |
| 数据库 | PostgreSQL | 15.8 |

| 日志系统 | Winston | 3.11.0 |

## 技术约束与规范

### 版本锁定
- 所有核心依赖严格锁定具体版本号
- 禁止使用 ^、~ 模糊版本匹配符号
- 必须生成并锁定 package-lock.json / yarn.lock 文件

### 防风控约束
- 所有自动化执行必须叠加全套仿真&指纹抹平能力
- 禁止裸机原生Playwright执行
- 所有浏览器实例必须加载内置防爬配置与真人行为逻辑

### 跨端统一约束
- 三端Windows/macOS/Ubuntu执行逻辑、延时规则、指纹参数、流程解析规则100%一致
- 无差异化打包、无平台专属特殊逻辑

### 安全约束
- 所有本地服务仅127.0.0.1本地监听,禁止外网暴露
- 租户数据严格隔离、接口全局鉴权
- 敏感数据脱敏、关键字段加密存储全覆盖

## 1. 执行摘要

**方案3核心策略**：优先完成Tauri桌面壳集成，确保三端一致性，然后并行推进后端核心模块和前端可视化后台开发。

**理由**：
- 桌面壳是系统的运行容器，提前完成可确保跨端一致性
- 后端核心模块（沙箱引擎、流程模板等）可独立开发
- 前端可视化后台与后端API紧密耦合，可在API就绪后快速开发
- 这种策略能最大程度降低跨端兼容性风险

**总工期**：8周

## 3. 技术规格详细说明

### 3.1 Tauri桌面壳技术规格

**核心要求**：
- Tauri版本：2.0.0-beta.28（固定）
- 所有本地服务仅127.0.0.1监听
- 禁止开启Tauri内置浏览器
- 集成独立Chromium内核路径配置

**桌面壳结构**：
```
src-tauri/
├── Cargo.toml                    # Rust依赖配置
├── tauri.conf.json               # Tauri配置
├── src/
│   └── main.rs                   # 桌面壳入口
└── icons/                        # 应用图标(三端)
```

**tauri.conf.json关键配置**：
- `security.csp`: 限制内容安全策略
- `plugins`: 禁用默认插件
- `windows`: 配置窗口大小和标题
- `bundle`: 配置三端打包参数

### 3.2 沙箱引擎技术规格

**核心组件**：

**延时模式配置**：

|------|-------------|-------------|------------|------------|

**失败重试机制**：
- 最大重试次数：3次（可配置）
- 重试间隔：指数退避（1s → 2s → 4s）
- 重试触发条件：元素定位失败、页面加载超时、网络异常

### 3.4 外部业务API接入层技术规格

**API配置项**：
- 请求地址(url)
- 请求方式(method): GET/POST
- 请求头(headers): Token、Content-Type等
- 请求参数(params): 查询参数
- 请求体(body): POST数据
- 数据列表路径(dataPath): 如 "data.list"
- 分页配置(pagination): page、pageSize、total
- 批量拉取数量(batchSize): 默认50

**数据映射机制**：
- 配置：`页面选择器 <===> 业务API字段Key`
- 示例：`input[name=phone] <===> phone`
- 支持：文本输入、下拉选择、单选按钮、复选框

**循环执行引擎**：
```javascript
async function runBatch(taskId: string, apiSourceId: string, sceneId: string) {
  const batchData = await apiClient.fetchBusinessData(apiSourceId);
  const total = batchData.length;
  
  for (let i = 0; i < total; i++) {
    const item = batchData[i];
    await sceneRunner.run(sceneId, item);
    await taskService.updateProgress(taskId, ((i + 1) / total) * 100);
  }
}
```

### 3.5 任务队列技术规格

**BullMQ配置**：
- 队列名称：`rpa-tasks`
- 并发数：根据租户配额动态调整
- 重试次数：3次（可配置）
- 重试延迟：指数退避
- 任务超时：30分钟（可配置）

**任务状态流转**：
```
pending → running → success/failed/cancelled
       ↘ (重试) ↗
```

**并发控制**：
- 按租户维度限流
- 全局最大并发数：10
- 超额任务自动排队

**前端技术栈**：

- Vue Router路由

**页面结构**：

| 风险编号 | 风险描述 | 概率 | 影响 | 优先级 | 缓解策略 |

| R-02 | 防风控策略失效 | 中 | 高 | P0 | 版本锁定，定期指纹检测，备用策略 |
| R-03 | 跨端兼容性问题 | 高 | 高 | P0 | 三端同步开发测试，统一代码逻辑 |
| R-04 | 内存泄露导致系统崩溃 | 中 | 高 | P0 | 进程池管理，资源强制回收，内存监控 |
| R-05 | 并发任务性能问题 | 中 | 中 | P1 | 并发限制，任务队列调度，性能测试 |
| R-06 | API数据映射错误 | 低 | 中 | P1 | 配置校验，映射预览，数据验证 |
| R-07 | 会话缓存失效 | 中 | 中 | P1 | 会话过期检测，自动重建，异常处理 |
| R-08 | 任务队列阻塞 | 低 | 高 | P1 | 队列监控，超时处理，死信队列 |
| R-09 | 数据库连接池耗尽 | 低 | 中 | P2 | 连接池配置，连接复用，监控告警 |
| R-10 | 安全漏洞（JWT泄露） | 低 | 高 | P1 | Token有效期限制，刷新机制，HTTPS |

### 6.2 关键风险缓解策略

#### R-01: Tauri v2 Beta版本稳定性问题
- **缓解措施**：
  - 严格锁定v2.0.0-beta.28版本，禁止升级
  - 建立回退机制，保存可用版本快照
  - 在三端分别进行完整测试
  - 关注官方发布日志，及时了解已知问题

#### R-02: 防风控策略失效
- **缓解措施**：
  - 锁定Playwright-Core 1.45.3版本
  - 建立指纹检测自动化测试
  - 保留多种指纹混淆方案作为备用
  - 定期更新指纹参数配置

#### R-03: 跨端兼容性问题
- **缓解措施**：
  - 统一代码库，无平台分支
  - 三端同步开发，同步测试
  - 使用统一的配置文件和参数
  - 建立跨端测试矩阵

#### R-04: 内存泄露导致系统崩溃
- **缓解措施**：
  - 任务结束强制销毁Chromium进程
  - 清空userDataDir和缓存
  - 定期监控进程内存使用
  - 设置内存上限告警

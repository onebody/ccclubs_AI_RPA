- **状态管理层**：`stores/execution.ts` 基于 Pinia 定义核心状态机，维护 `step`（如 `checking_login`, `qr_scanning`）、`companies` 及 `taskId`，并暴露 `startExecution`、`doSelectCompany` 等动作。
- **视图层**：`components/ExecutionPanel.vue` 作为展示组件，通过 `v-if/v-else-if` 响应 `executionStore.step` 的变化，渲染二维码、单位列表或进度提示。
- **通信层**：`api/ws.ts` 封装单例 WebSocket 客户端，负责接收后端推送的实时消息（如 `qr_code`, `company_list`）并转发给 Store；`api/execution.ts` 提供 RESTful 接口用于用户主动触发的操作（如扫码完成、取消任务）。
- **数据流**：WebSocket 消息驱动 Store 状态更新，进而触发 UI 重绘；用户交互通过 Store 动作调用 API 或直接修改本地状态（演示模式）。

- **分层架构**：采用经典的 API-Service-Browser 三层结构。`app/api` 定义 REST 接口与 WebSocket 端点；`app/services` 封装业务逻辑（如 `TaskService`）并协调异步执行；`app/browser` 负责底层 Playwright 自动化操作与会话管理。
- **并发模型**：为解决 Playwright Sync API 与 FastAPI Async 环境的冲突，`BrowserSessionManager` 维护了一个专用的后台工作线程（Worker Thread），通过 `queue.Queue` 接收主线程下发的自动化指令，确保浏览器操作的线程安全性。
- **人机协同机制**：通过 `ExecutionWaiter` 和 WebSocket (`ws/manager.py`) 实现阻塞式等待。在扫码登录、单位选择等关键节点，后端挂起自动化流程并向客户端推送状态，待用户手动干预后恢复执行。
- **状态持久化**：利用 `data/browser_states/` 目录下的 JSON 文件按省份存储 `storage_state`，实现跨任务的登录态复用，减少重复登录频率。
